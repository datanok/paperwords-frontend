# Product

## Register

product

## Users

Friends and family passing a phone or tablet around a table. One person picks a secret word; the other guesses it based only on a letter-match count per guess. The game is social and informal — played on a couch, at a café, on a bus. No accounts, no server, no setup friction.

## Product Purpose

Word Hunt is a local, pass-the-phone word-guessing game built on a handcrafted paper aesthetic. Two players take turns using one device. Success means the guesser deduces a hidden word purely from numeric letter-match hints. The experience should feel like scribbling a game on a piece of notebook paper — charming, low-stakes, instantly understood.

## Brand Personality

Warm, handcrafted, playful-but-not-childish. Feels like a notebook doodle, not an app.

## Anti-references

- Wordle / NYT Games: clean digital tile grids, rounded sans, flat pastels — avoid
- Kids apps / cartoon aesthetics: bright crayon colors, mascots, bubble fonts — avoid
- Casino / puzzle apps: heavy gradients, glow effects, stars-and-badges chrome — avoid
- Generic mobile games: dark mode, neon, app-store template feel — avoid

## Design Principles

1. **Paper over pixels** — every surface should feel like it was drawn on a notebook page, not rendered on a screen. Sketchy SVG filters, slight rotations, grid backgrounds.
2. **Consistent card grammar** — every game phase uses the same card treatment: cream background, sketchy inked border, a small tilt. No screen should feel like it belongs to a different game.
3. **Hand-drawn restraint** — ornamentation (stars, squiggles, tallies) earns its place by communicating game state, not decorating. One or two hand-drawn accents per screen, not seven.
4. **Legible over stylish** — the handwritten fonts carry personality; the monospace carries data. Never sacrifice legibility for flair.

## Accessibility & Inclusion

- Minimum WCAG AA contrast for all interactive text
- Hidden input pattern for letter entry (visible dots, invisible native input) must remain keyboard-navigable
- `autoFocus` on each phase's primary input to reduce tap friction on mobile
